# PCB Placement Requirements — EE-Grade Layout Rules

These rules apply to ALL boards generated by the pipeline, not just nl-s-3c.
The placement engine must think like a skilled EE, not a bin-packing algorithm.

## 1. Voltage Domain Isolation

- **24V power section**: Isolated group, near power input connector. All 24V
  components (buck converter, input caps, protection diodes) together.
- **5V relay section**: Adjacent to 24V side (shares power path), but
  separated. Each relay and its driver circuit as a unit.
- **3.3V digital section**: Own group, centered around MCU.
- **Analog section**: Isolated from digital, own ground region if possible.
- Ground plane splits or star-point connections between voltage domains.

## 2. Functional Sub-Circuit Clustering

Components that form a sub-circuit MUST be placed as a tight unit:
- **Relay driver**: Q (transistor) adjacent to its relay (K), with flyback
  diode (D) across relay coil pins, gate resistor (R) near Q gate, and
  indicator LED + series resistor nearby. All oriented for short traces.
- **Buck converter**: Input cap → IC → inductor → output cap in signal flow
  order. Feedback divider resistors near FB pin.
- **Voltage regulator**: Input cap → regulator → output cap, linear flow.
- **Decoupling caps**: Within 3mm of the power pin they protect. MUST be
  closer to the IC than any other component.
- **Crystal/oscillator**: Within 5mm of MCU clock pins, with load caps
  immediately adjacent.

## 3. Connector Placement

- **Ethernet (RJ45)**: Board edge, with PoE components directly behind it.
  Ethernet PHY/driver IC between connector and MCU.
- **USB**: Board edge.
- **SD card**: Board edge.
- **Screw terminals / power input**: Board edge, near power section.
- **Programming/debug headers**: Board edge or accessible location.
- Connectors oriented so cables route away from board, not across it.

## 4. Signal Flow Ordering

Components should be placed in signal flow order:
- Power: Input connector → protection → converter → output → load
- Data: Connector → PHY/driver → MCU → peripherals
- Relay: MCU GPIO → gate resistor → transistor → relay → terminal

## 5. Pin Orientation

- Passive components (R, C) oriented so their pads align with the trace
  direction to their connected pins (minimize vias and bends).
- ICs oriented so high-speed/critical pins face their destinations.
- Connectors oriented so pin 1 faces the interior signal path.

## 6. Component Spacing

- SMD passives: minimum 0.5mm courtyard gap.
- THT components: minimum 1.0mm courtyard gap.
- Between voltage domains: minimum 2.0mm clearance.
- High-voltage (>50V) to low-voltage: per IPC-2221 clearance.

## 8. Review Criteria (What an EE Would Check)

An automated review agent should verify:
1. No component collisions (courtyard overlap)
2. All decoupling caps within 3mm of their IC power pins
3. Sub-circuit components clustered (relay+driver, buck converter chain)
4. Voltage domain separation maintained
5. Connectors on board edges with correct orientation
6. Signal flow is logical (no long traces crossing the board)
7. Thermal: high-power components not adjacent to temperature-sensitive parts
8. Crystal close to MCU with short, matched traces

## 9. Optimization Strategy — 3-Level Hierarchical Pipeline (v5)

Do NOT use random perturbation (SA nudge/swap). Do NOT use 15+ sequential
phases that fight each other. Instead, use a **top-down hierarchical approach**:

### Level 1 — Zone Partitioning (`zone_partitioner.py`)
Partition the board into non-overlapping rectangular zones using keyword
matching on FeatureBlock names:
- `power`, `relay`, `mcu`, `analog`, `ethernet`, `display`
- Default proportions from reference board analysis
- Zones scaled by component count, with 5mm inter-zone gaps
- Each FeatureBlock maps to exactly one zone

### Level 2 — Group Placement (`group_placer.py`)
Place each FeatureBlock as a **rigid rectangular unit** within its zone:
- Internal component offsets from `_layout_group()` are preserved
- Groups are placed largest-first using collision-free grid search
- Connectors within groups are slid to the nearest board edge
- Ferrites at isolation boundaries are placed at zone edges
- **No phase fights**: groups never mix between zones

### Level 3 — Intra-Group Refinement (`placement_optimizer.py`)
Fine-tune within groups (single pass, no iteration):
- **3a** Relay row formation (K refs in 1xN horizontal row)
- **3b** Relay driver subgroup tightening (Q+D+R within 8mm of K)
- **3c** Decoupling cap tightening (within 3-5mm of IC)
- **3d** Crystal-MCU proximity (within 10mm)
- **3e** RF edge pinning (antenna modules to board edge)
- **3f** Connector orientation (face outward from nearest edge)
- **3g** Collision resolution (group-constrained, then unconstrained)

### Scoring (12 dimensions)
- Collisions (20%), Voltage Isolation (15%), Subcircuit Cohesion (5%),
  Connector Edge (10%), Decoupling Proximity (10%), MCU Peripheral (10%),
  RF Edge (5%), Connector Orientation (5%), Regulator Boundary (5%),
  Group Cohesion (5%), Subgroup Cohesion (5%), Group Isolation (5%)

### Visual Feedback
Renders at each level for debugging:
- `output/placement_L1_zones.png` — zone rectangles only
- `output/placement_L2_groups.png` — groups as colored boxes
- `output/placement_groups.png` — final component-level render

Each level is complete before the next starts. No level undoes work
from a previous level. The review agent gives the same critique a
senior EE would give.
